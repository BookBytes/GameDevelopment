Introduction to Game Development

Artificial Intelligence

What is AI?

  • The study of computational systems that "exhibit intelligence"
  • Capabilities: problem solving, planning, and learning
  • Practices: studying behavior and processing (human, rational, and cognitive), machine learning, searching, optimization, knowledge bases

AI in Games

  • Opponents (e.g., how can they outsmart you?)
  • Teammates
  • Support characters
  • Autonomous characters
  • Commentators
  • Camera control
  • Plot
  • Programming usually done in LISP or PROLOG languages
  • In short: essential to the modeled world
  • Goals of AI in games:
    • Provide challenging opponents...
    • ...but not too challenging or too predictable
    • Gather enough knowledge...
    • ...but aim for low CPU and memory usage
    • Provide a perception of reality (e.g., human characteristics)
    • Conflicting goals eh?

Effects of Bad AI (duh!)

  • Reduced fun
  • Extreme behaviors (from lame to superhuman)
  • Unrealistic behaviors
  • Mindless hordes (of opponents)
  • Cheating

Sample Problems

  • What to do when the enemy is heard or sensed?
  • What to do when my health level is low?
  • What to do if don't see or hear anything?
  • How to move from point A to the goal in a maze?
  • Identify a good day for a round of golf
  • How to move a flock of objects (e.g., birds)?

Decision-Making Techniques

  • Finite State Machines
  • Search
  • Path-Finding
  • Decision Trees
  • Neural Networks
  • Genetic Algorithms
  • Fuzzy Logic
  • and many more...

Building an AI Agent

  • Given: the environment
  • Cycle:
    • Sense - Gather information about the environment
    • Think - Use one of the decision-making techniques above
    • Act - Execute based on your decision
    • Repeat steps
  • Share global data each agent structures and maps
  • Each time through game loop, "tick" each agent
  • Constraints:
    • Memory: what to remember and for how long?
    • Obstacles (can be anything)
    • Complex and differing structures and sources in your game (e.g., sound, vision, etc)
    • Knowledge intensive
    • CPU usage

Finite State Machines (FSM)

  • Simplest of AI engines (and to understand)
  • Almost exactly like the state diagrams in game design documents
  • A state has behavior and trigger
  • Strengths:
    • Simple, quick, sufficient
    • Tools available for non-programmers
    • Can be compacted into a simple data structure, including an array
  • Weaknesses: number of states can grow very fast (alas, best to keep it simple)
  • FSM Example: Quake II
    • 9 different states:
      • Standing
      • Walking
      • Running
      • Dodging
      • Attaching
      • Melee
      • See the enemy
      • Idle
      • Searching

A*

  • Path finding algorithm
  • The idea: want to go from point A to point B; keep track of the cost from getting from point to point; will get the least cost to get from point A to point B
  • Strengths:
    • Popularity
    • Very configurable for different game types and maps
  • Weaknesses:
    • Memory-hungry (i.e., lots of thinking)
    • Very slow on rough terrains
    • Largely inefficient for real-time robotics
  • (Java) implementation and example: http://www.cokeandcode.com/main/tutorials/path-finding/

Minimax Trees

  • The idea: look ahead, predicting what the opponent might do
  • Mini and max - "minimize the opponents' maximum move
  • One input: depth
  • Recursive
  • For even a simple game, a tree can have a million nodes (impossible to analyze the whole board)
  • Helper functions:
    • Move generator - determine what moves are possible
    • Static evaluator - Figure out how good each move is, based on changes in board position
  • Guess what board game uses this algorithm?
